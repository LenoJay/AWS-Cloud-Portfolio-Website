How the architecture works

User request flow (runtime)

  1. User visits your domain (example: lenojay.com).
  2. DNS routes the request to CloudFront.
  3. CloudFront serves cached files or fetches them from the S3 origin.
  4. The S3 bucket stays private; CloudFront is the only service that reads from it.

Deployment flow (updates)

  1. You push changes to GitHub.
  2. GitHub Actions runs a deploy workflow.
  3. The workflow uploads updated site files to S3, and the site updates publicly.

Phases (what you will do)

  1. Repo setup (GitHub + VS Code)
  2. Build the static site locally (HTML/CSS/JS + folder structure)
  3. Create the S3 origin bucket + upload the site files
  4. Register your domain + hosted zone (Route 53)
  5. Request certificate (ACM)
  6. Create CloudFront distribution + connect origin + DNS
  7. CI/CD (GitHub Actions deploy)
